Dataset¶

I used the dataset from previous homework - pc1 dataset from openml that analyses the defects in the software. I preprocessed it using StandardScaler and LabelEncoder for the labels to be in [0, 1] set. I also split the dataset into train and test sets with test set consisting of 33% of the data.

Models¶

I trained 4 models:

  • RandomForest with default hyperparameters (my primary model)
  • XGBoost with default hyperparameters (first compare model)
  • LogisticRegression with default hyperparameters (second compare model)
  • RandomForest with n_estimators=10, max_depth=10 which I later call SmallRandomForest (third compare model)

PVI for RandomForest¶

Below I present the PVI plot for the RandomForest model:

The most important feature is the uniqOpnd followed by IOBlank and I. In the dataset these variables are associated with respectively number of unique operands in the code, number of blank lines and the length of the code. It is rational that when writing code these variables are significant when it comes to vulnerability analysis. We may also hypothesise that these variables are correlated as it makes sense that with higher number of lines naturally comes a higher number of blank lines of codes and number of operands, so let's look at the correlation matrix between variables that are present in the PVI plot above:

Indeed we can see that these variables are correlated with correlation values being:

  • 0.78 for correlation between uniq_Opnd and lOBlank
  • 0.72 for correlation between uniq_Opnd and I
  • 0.49 for correlation between lOBlank and I

RandomForest vs SmallRandomForest¶

Let's first start with the comparison of RandomForest and SmallRandomForest. The first 4 variables are the same which is not surprising as the models share the same architecture with different hyperparameters. The smaller model however emphasises more the importance of uniq_Opnd suggesting that larger RandomForest can better distinguish the nuances between variables and focus on impact of other ones.

RandomForest vs XGBoost¶

In this case XGBoost focuses on a different most important variable namely I - length of the code. Second most important variable is the same in both models. Interestingly the third most important variable in XGBoost is uniqOp - number of unique operators which was not present in top 10 most important variables for RandomForest.

RandomForest vs LogisticRegression¶

Here again UniqOpnd is the most important variable in both models however there are more differences in other variables - third most important variable in LogisticRegression is uniq_Op which is not present among top 10 variables in RandomForest and the second most important variable - D in LogisticRegression is the last in RandomForest.

Comments¶

As we could have seen previously many variables are correlated, so tree-based models better capture the nuances between the variables, however it is interesting that both LogisticRegression and XGBoost focus a lot on uniq_Op whereas RandomForest does not.

Comments¶

PVI and SHAP yield very similar results - the top 3 most important variables are the same in both cases: uniq_Opnd, lOBlank and I. More differences appear in Gini as even though I and uniq_Opnd are in top 3 variables, the first place goes to lOComment (count of lines of comments) which is not considered as an important variable in other methods (not present in top 10 in PVI and 13th in SHAP). All methods have different working mechanism so it is not that surprising that they give different results, however it is interesting that Gini focuses on lOComment which is not present in other methods. We can also see that loComment is correlated with uniq_Opnd and lOBlank so therefore Gini that looks at the impurity of the node maybe focuses more on this variable. from sklearn.linear_model import LogisticRegression
from sklearn.ensemble import RandomForestClassifier
import dalex as dx
from xgboost import XGBClassifier


def dx_predict_proba(model, df):
    return model.predict_proba(df)[:, 1]


RANDOM_STATE = 42

models = {
    'RandomForest': RandomForestClassifier(random_state=RANDOM_STATE, n_jobs=-1),
    'XGBoost': XGBClassifier(random_state=RANDOM_STATE, n_jobs=-1),
    'LogisticRegression': LogisticRegression(random_state=RANDOM_STATE, n_jobs=-1),
    'SmallRandomForest': RandomForestClassifier(n_estimators=10, max_depth=10, random_state=RANDOM_STATE, n_jobs=-1),
}

explainers = {}

for model_name, model in models.items():
    print(f'Fitting {model_name}...')
    model.fit(X_train, y_train)
    print(f'{model_name} predictions')
    explainer = dx.Explainer(model, X_test, y_test, predict_function=dx_predict_proba, label=model_name)
    explainers[model_name] = explainer
